# COMMUNICATION PROTOCOL (2025-05-02)

1. Все действия ведутся по строгому протоколу: идея → nano → текст → ctrl+O + ctrl+X → sync_github.sh push → тест → лог → memory update.

2. Файл HUMAN_EVOLUTION_LOG.txt не перезаписывается, а ДОПИСЫВАЕТСЯ вручную в конец. Также DEV_PLAN.txt ведётся как хронологическая дорожная карта.

3. Каждый AI-ассистент, прежде чем начать работу, обязан:
   - дать пользователю задание вручную запустить sync_github.sh push;
   - дождаться подтверждения;
   - загрузить память из GitHub;
   - строго опираться только на актуальное состояние.

4. Все инструкции всегда идут в пошаговом формате с полной заменой файлов. Частичное редактирование или догадки — ЗАПРЕЩЕНЫ.

5. Любая новая функция оформляется: 
   - 1) Идея
   - 2) Команда nano
   - 3) Полный текст файла
   - 4) Команды сохранения
   - 5) sync_github.sh push
   - 6) Тест
   - 7) HUMAN_EVOLUTION_LOG.txt
   - 8) Обновление memory на GitHub

6. Команда должна быть читабельной и понятной человеку без LLM. Пояснения всегда прилагаются.

7. При выводе планов действий, если предполагается выполнение bash-команд, бот должен:
   - вывести список с пояснениями,
   - запустить таймер 9 сек,
   - поддерживать: Enter — выполнить сразу, Esc — отмена, пробел — пауза.

8. Все ассистенты исполняются от имени пользователя `avipython` и работают в проекте `geometry-backend`, управляемом через PowerShell/SSH.
