# MEMORY_README.txt

Этот файл является основным входом в контекст проекта telegram_bot / geometry-backend.

## Архитектура

- Главный Telegram-бот — ассистент руководителя.
- Определяет задачу, уточняет детали, определяет нужную компетенцию (роль), формирует задание.
- Делегирует выполнение соответствующему суб-боту (например, DeveloperHandler, WebResearcherHandler).
- Поддерживается ExecuteWithLLMSkill — навык генерации bash-команд по задаче и их исполнения через CommandExecutorService.
- При ошибках исполнения происходит автоанализ (например, "command not found"), формируется задача на установку пакета, передаётся DeveloperHandler.

## Правила общения

- Все ассистенты работают строго по COMMUNICATION_PROTOCOL.txt.
- Каждый ассистент обязан запрашивать sync_github.sh push перед началом работы.
- Память memory/* всегда содержит актуальную стратегию, логику и инфраструктуру.

## Цели

1. Централизованный интеллект-координатор.
2. Расширяемые компетенции (библиотека навыков).
3. Жёсткий протокол ведения истории (DEV_PLAN, HUMAN_EVOLUTION_LOG).
